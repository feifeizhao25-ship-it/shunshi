# SEASONS — Skills System (Developer Guide)

> SEASONS does not ship a large static content library. Professional wellness content is dynamically generated by AI Skills powered by a large language model.
> This document covers the 12 built-in Skills: their purpose, input/output schema, and how to extend the system.

---

## Overview

Skills are self-contained units of AI-powered functionality. Each Skill:
- Has a unique `code` (e.g., `daily_rhythm_plan`)
- Is triggered by user actions or contextual events
- Returns a structured `OutputSchema` (text + cards + follow-up)
- Can be free or premium (`is_premium: bool`)
- Has a `caching_strategy` to reduce API calls

Skills are **not** Flutter Riverpod providers. They run on the backend and are invoked via the AI chat endpoints (`/ai/chat`). The Flutter app sends user context and the backend dispatches to the appropriate Skill.

## Skill Detail

### 1. DailyRhythmPlan (`daily_rhythm_plan`)

**Purpose:** Generate a personalized daily wellness plan for the user.

**Inputs:**
```json
{
  "user_id": "usr_abc123",
  "context": {
    "life_stage": "young_adult",
    "today_status": "a_bit_tired",
    "current_solar_term": "spring_equinox"
  }
}
```

**Behavior:**
1. Fetches user's habit history and current season
2. Builds a prompt including user context + season + habits
3. Returns a structured daily plan with greeting, insight, three actions, habit checklist, and AI care note

**Output Example:**
```json
{
  "text": "Good morning, Jane. Today feels like a gentle day — slow down a little and let the spring light guide you.",
  "cards": [],
  "follow_up": ["Did you manage the morning walk yesterday?"],
  "care_status": "good"
}
```

**Caching:** `short` — cache for 1 hour. A user's daily plan doesn't change mid-day.

### 2. MoodFirstAid (`mood_first_aid`)

**Purpose:** Immediate emotional support when user shares difficult feelings.

**Inputs:**
```json
{
  "user_id": "usr_abc123",
  "context": {
    "emotion": "anxious",
    "intensity": 7,
    "message": "I can't stop worrying about my presentation tomorrow"
  }
}
```

**Behavior:**
1. Classifies emotion and intensity
2. Checks for high-risk signals (`emotion in ["hopelessness", "suicide", "self_harm"]` or `intensity >= 9`)
3. If high-risk: triggers SafeMode — a special response directing user to professional help
4. Otherwise: generates an empathetic response with one small, actionable suggestion

**High-Risk Flow:**
```json
{
  "text": "I'm really glad you reached out. What you're describing sounds very hard right now...",
  "care_status": "critical"
}
```

**Caching:** `none` — always fresh emotional support.

**Design Rules:**
- Never say "you are [label]"
- Never say "you've been [pattern]"
- Only state observations, never judgments
- Always offer a tiny action (not a long plan)

### 9. FollowUpGenerator (`follow_up_generator`)

**Purpose:** System skill that generates a gentle follow-up message to send to the user a few days after a conversation.

**Inputs:**

**Rules:**

- Must be easy to ignore (no pressure)
- Can be: care check-in, gentle reminder, or a seasonal tip sharing

### 10. PresencePolicyDecider (`presence_policy_decider`)

**Purpose:** Decide how actively the AI should reach out to the user.

**Inputs:**
```json
{
  "user_id": "usr_abc123",
  "context": {
    "interaction_frequency": "low",
    "user_settings": {
      "notifications_enabled": true
    }
  }

```

**Output:** One of four presence levels:
- `quiet` — only respond when user initiates
- `light` — one follow-up per week
- `engaged` — 2–3 check-ins per week
- `active` — daily check-ins

**Caching:** `long` — recalculated weekly.

## Adding a New Skill

### 1. Create the Skill class

```python
# backend/ai/skills/seasonal_reflection.py
from .base import BaseSkill, InputSchema, OutputSchema, SkillCategory

class SeasonalReflectionSkill(BaseSkill):
    code = "seasonal_reflection"
    name = "Seasonal Reflection"
    category = SkillCategory.WELLNESS
    description = "Generate a reflective exercise aligned with the current season."
    is_premium = False

    async def execute(self, params: InputSchema) -> OutputSchema:
        current_season = params.context.get("season", "spring")
        # ... generate reflection exercise ...
        return OutputSchema(
            text="...",
            cards=[...],
            follow_up=["..."],
            care_status="good"
        )

    def get_caching_strategy(self) -> str:
        return "long"  # One per season
```

### 2. Register it

```python
# backend/ai/skills/registry.py
from .seasonal_reflection import SeasonalReflectionSkill

SkillRegistry.register(SeasonalReflectionSkill())
```

### 3. Document it

Add the skill to this document: code, name, category, purpose, inputs, output example, caching strategy.

## Skill Invocation from Flutter

Skills are invoked via the standard chat API. The Flutter app sends context that triggers the appropriate Skill on the backend:

```dart
// Triggering SolarTermGuide from Flutter:
final response = await ApiService.instance.sendChat(
  prompt: "What's the guidance for the Spring Equinox?",
  userId: userId,
);
```

The backend interprets the prompt and dispatches to `SolarTermGuideSkill`. The Flutter app renders the response using the same chat UI — `ChatResponse.text` and `ChatResponse.suggestions`.

## SafeMode (Crisis Response)

`MoodFirstAid` triggers `SafeMode` when high-risk signals are detected:

1. AI response is replaced with a SafeMode card
2. `care_status` is set to `"critical"`
3. UI shows a prominent card: "I'm really glad you reached out. You're not alone."
4. Suggested actions: crisis hotline, contact trusted person, schedule therapy appointment

SafeMode card design:
- Non-alarming, warm visual tone
- Clear next-step suggestions
- Never dismisses the user's feelings
