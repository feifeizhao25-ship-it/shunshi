## 三、AI 陪伴系统（核心）

### 3.1 定位
顺时的 **AI 灵魂**，是最核心的能力。所有其他系统都围绕它服务。

### 3.2 核心能力矩阵

| 能力 | 描述 | 技术实现 |
|------|------|----------|
| **Life Memory** | 长期记忆用户偏好、习惯、家庭 | Vector DB + Graph DB |
| **Care Status** | 实时感知用户状态（情绪/精力/健康） | 多维度打分 + 时序分析 |
| **Follow-up** | 主动跟进之前聊过的话题 | 待办队列 + 定时触发 |
| **Presence Reduction** | 知道何时不打扰用户 | 智能触达引擎 |
| **Context Depth** | 记住用户几个月前说过的话 | 记忆检索增强 |

### 3.3 对话示例

```
【探索期用户】
用户：最近睡不着
顺时：又熬夜啦？😊 是不是又在刷手机？
用户：嗯...
顺时：把手机放远点？实在睡不着来找我聊天~

【压力期用户】
用户：今天又被领导骂了
顺时：上班不容易。先喝杯水休息一下？
       你已经很努力了。
       最近睡眠怎么样？

【陪伴期用户】
顺时：早上好！今天天气不错，可以出去走走。
用户：好，谢谢
顺时：不客气。记得中午吃药哦。
```

## 四、节律系统（核心差异）

### 4.1 定位
顺时与所有健康 App 的**根本差异**：
- 竞品：健康监测（给你数据）
- 顺时：生活节律管理（给你洞察）

### 4.2 四大节律模块

| 模块 | 分析内容 | 洞察示例 |
|------|----------|----------|
| **睡眠节律** | 入睡/起床时间、睡眠时长、质量趋势 | 「最近你睡得越来越晚，身体有点累了」 |
| **饮食节律** | 用餐时间、饮食结构、宵夜频率 | 「你最近吃宵夜有点多哦」 |
| **运动节律** | 活动时段、久坐提醒、运动坚持 | 「这周运动有点少，要不要动动？」 |
| **情绪节律** | 情绪波动、压力来源、社交活跃 | 「最近压力有点大，我陪你说说话？」 |

## 五、健康系统（40岁+核心）

### 5.2 五大健康模块

| 模块 | 目标用户 | 核心功能 |
|------|----------|----------|
| **体质调理** | 40-60岁 | 九种体质辨识、个性化调理方案 |
| **节气养生** | 全年龄段 | 24节气养生建议、食疗方案 |
| **食疗方案** | 全年龄段 | 药食同源推荐、禁忌提醒 |

## 九、数据系统（万亿护城河）

### 9.1 数据展示原则

```
❌ 不给用户看                          ✓ 给用户看
────────────────                      ─────────────────
「睡眠评分 62」                       「最近睡得越来越好~」
「今日步数 8,532」                    「今天动了动，身体棒」
「心率 78bpm」                        「心跳很正常」

核心原则：
• 只给洞察，不给数据
• 只说结论，不列数字
• 正面引导，不制造焦虑
```

## 十一、10年产品路线图

### Phase 1: 基础构建（Year 1-2）
- Q1-Q2: MVP - 基础对话、简单记忆、免费版
- Q3-Q4: V1.0 - AI Router、Life Stage、尊享版、1万条内容
- **目标**: DAU 10,000 | 次日留存 40% | 付费率 3%

### Phase 2: 增长突破（Year 3-5）
- Year 3: 家庭系统 - 绑定、查看、关怀提醒、Apple Watch
- Year 4: 内容生态 - 5万条内容、UGC社区
- Year 5: 多端覆盖 - 手表、音箱、家庭套餐
- **目标**: MAU 1,000,000 | 家庭绑定率 30% | 付费率 10%

### Phase 3: 生态扩张（Year 6-10）
- Year 6-7: 医疗对接 - 设备数据、家庭医生、健康预警
- Year 8-9: 车机覆盖 - 主流车机、驾驶陪伴
- Year 10: 终极形态 - 100万+内容、全球化
- **目标**: MAU 10,000,000+ | 家庭用户 50% | 年收入 10亿+
