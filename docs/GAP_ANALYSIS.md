# 顺时项目未考虑点补全清单

> 从"做完"到"做好"的20个关键缺口

## 一、产品层补全

### 1.1 首次用户阶段判断机制

| 项目 | 内容 |
|------|------|
| 问题 | 第一次没有数据时，如何判断用户属于哪个人生阶段？ |
| 风险 | 判断错了，第一印象就差了 |
| 解决方案 | - 首次轻问卷机制（3-5题）<br>- 行为反推机制（使用1周后自动校正）<br>- 用户手动校正入口<br>- 阶段"可漂移"而非固定 |
| 技术要求 | 用户表增加 `initial_stage_source` 字段，记录来源 |
| 优先级 | P0 |
| 不做的后果 | 大量用户首页不对，流失率高 |

### 1.2 AI表达克制规则

| 项目 | 内容 |
|------|------|
| 问题 | AI说得太准会让用户不舒服，"你最近很压抑"这类表达需要克制 |
| 风险 | 用户觉得被"看穿"，产生防备心理 |
| 解决方案 | - 记忆呈现克制规则<br>- 不下结论规则<br>- "只说观察，不说判断"的表达体系<br>- 负面标签禁止输出 |
| 技术要求 | Prompt中增加"表达克制"约束，测试集增加"舒适度"维度 |
| 优先级 | P0 |
| 不做的后果 | 用户感觉被冒犯，品牌信任受损 |

### 1.3 免费用户付费边界

| 项目 | 内容 |
|------|------|
| 问题 | 免费用户体验什么程度才会愿意付费？边界在哪？ |
| 风险 | 免费太弱直接走，免费太强付费感知不明显 |
| 解决方案 | - 免费7天完整体验（包含1次深度对话）<br>- 付费转折点：连续照护、长期记忆、家庭安心<br>- 免费用户能感知"被记住一点点"<br>- 免费不显示趋势图，只显示"今天" |
| 优先级 | P1 |
| 不做的后果 | 转化率低，付费增长慢 |

### 1.4 首页动态切换的稳定性

| 项目 | 内容 |
|------|------|
| 问题 | 首页变化太多，用户会失去熟悉感 |
| 风险 | 用户觉得"不会用了" |
| 解决方案 | - 固定导航骨架（5个Tab不变）<br>- 生命周期只改模块权重，不改主路径<br>- 首页模块渐进变化，而非突变 |
| 优先级 | P1 |
| 不做的后果 | 用户熟悉度下降，DAU受影响 |

## 二、AI系统层补全

### 2.1 模型人格一致性

| 项目 | 内容 |
|------|------|
| 问题 | 7B和72B说出来的话像两个人，付费切换时会割裂 |
| 风险 | 升级时觉得"变人了"，掉级时觉得"被降级" |
| 解决方案 | - 统一风格后处理层<br>- 语气一致性测试集<br>- 模型差异只在"深度"，不体现在"人格" |
| 技术要求 | 增加"人格一致性"测试维度 |
| 优先级 | P0 |
| 不做的后果 | 付费用户体验割裂，投诉增加 |

### 2.2 Skills输出稳定性

| 项目 | 内容 |
|------|------|
| 问题 | 同类建议今天和明天生成可能不一致 |
| 风险 | 用户觉得AI不靠谱 |
| 解决方案 | - 每个skill输出长度上限<br>- 每个skill表达模板统一<br>- 禁忌字段强制存在<br>- "内容稳定性"回归测试 |
| 技术要求 | 测试系统增加"一致性"维度 |
| 优先级 | P0 |
| 不做的后果 | 用户信任度下降 |

### 2.3 上下文污染控制

| 项目 | 内容 |
|------|------|
| 问题 | 上下文越多，模型越容易被带偏 |
| 风险 | AI越来越啰嗦、越来越像在"表演懂" |
| 解决方案 | - 长期上下文 vs 临时上下文分离<br>- 同时投喂的字段限制<br>- 上下文摘要压缩规则<br>- 定期清理过期上下文 |
| 优先级 | P1 |
| 不做的后果 | AI回答质量随时间下降 |

### 2.4 长期关系评测

| 项目 | 内容 |
|------|------|
| 问题 | 当前测试是单轮，缺少连续7天/30天/90天的关系测试 |
| 风险 | 长期使用后AI关系可能不稳定 |
| 解决方案 | - 长链路关系测试<br>- 多轮记忆一致性测试<br>- 退让机制长期效果测试<br>- 付费用户长期满意度自动评估 |
| 优先级 | P2（上线后迭代） |
| 不做的后果 | 长期用户流失率高 |

## 三、内容与技能层补全

### 3.1 媒体资产策略

| 项目 | 内容 |
|------|------|
| 问题 | 不做大内容库，但用户还是期待看图/视频 |
| 风险 | 产品"很聪明，但总是没图" |
| 解决方案 | "小而精的媒体资产库"策略：<br>- 穴位位置图（必须）<br>- 动作演示（必须）<br>- 食谱示意图（半自动）<br>- 八段锦镜像（必须） |
| 优先级 | P1 |
| 不做的后果 | 用户体验不完整 |

### 3.2 用户个人资产库

| 项目 | 内容 |
|------|------|
| 问题 | 用户能否把顺时用成自己的生活工具箱？ |
| 风险 | 用户留存只靠AI对话，无"资产"沉淀 |
| 解决方案 | - 用户收藏体系<br>- 个人常用内容库<br>- 家庭共享内容<br>- "我的茶饮"、"我的穴位"个人中心 |
| 优先级 | P2 |
| 不做的后果 | 留存依赖单一，卸载后无牵挂 |

## 四、家庭系统补全

### 4.1 家庭角色模型

| 项目 | 内容 |
|------|------|
| 问题 | 家庭不只是"子女→父母"，还有伴侣、兄弟姐妹 |
| 风险 | 真实家庭场景覆盖不全 |
| 解决方案 | - 家庭角色模型：子女/父母/伴侣/兄弟姐妹<br>- 多照护者权限<br>- 主照护人机制<br>- 家庭责任冲突边界 |
| 优先级 | P1 |
| 不做的后果 | 家庭场景覆盖不全 |

### 4.2 反监控系统设计

| 项目 | 内容 |
|------|------|
| 问题 | 家庭摘要容易变成"监控"，从安心变担心 |
| 风险 | 父母觉得被监视，家庭矛盾 |
| 解决方案 | - 父母知情与同意机制<br>- 可关闭家庭共享入口<br>- 家庭成员可见信息分层<br>- 紧急状态才升级 |
| 优先级 | P0 |
| 不做的后果 | 家庭投诉，法律风险 |

## 五、商业与增长层补全

### 5.1 礼品场景

| 项目 | 内容 |
|------|------|
| 问题 | "送给父母的礼物"是很大潜力，但还没展开 |
| 风险 | 错过一个高转化场景 |
| 解决方案 | - 节日送礼入口<br>- 代父母开通流程<br>- 礼物文案体系<br>- 礼物到期续费逻辑 |
| 优先级 | P2 |
| 不做的后果 | 付费转化少一个重要来源 |

### 5.2 企业场景

| 项目 | 内容 |
|------|------|

| 风险 | 错过企业市场 |
| 解决方案 | - B端后台<br>- 企业版权益<br>- 匿名聚合报告<br>- 企业不接触个人隐私的边界设计 |
| 优先级 | P3（长期） |
| 不做的后果 | 市场少一个分支 |

### 5.3 套餐升级路径

| 项目 | 内容 |
|------|------|
| 问题 | 用户从个人升级到家庭，历史如何延续？ |
| 风险 | 升级时体验断层 |
| 解决方案 | - 套餐迁移路径设计<br>- 权益可继承<br>- 历史记忆延续<br>- 升级过渡页设计 |
| 优先级 | P2 |
| 不做的后果 | 升级体验差 |

## 七、国际版与扩展层补全

### 7.1 海外记忆权限

| 项目 | 内容 |
|------|------|
| 问题 | 海外用户对"被记住"更敏感 |
| 风险 | 默认开启可能违反隐私预期 |
| 解决方案 | - 国际版默认轻记忆<br>- 用户主动选择深度记忆<br>- 记忆权限独立设置页 |
| 优先级 | P1（国际化时） |
| 不做的后果 | 海外隐私投诉 |

### 7.2 设备优先级

| 项目 | 内容 |
|------|------|
| 问题 | 手表/车机/音箱先接哪个？ |
| 风险 | 什么都接什么都不强 |
| 解决方案 | - 手机是主脑<br>- 手表是轻提醒/轻动作<br>- 音箱是老年语音入口<br>- 车机是通勤节律场景 |
| 优先级 | P3 |
| 不做的后果 | 设备体验都不强 |

### 8.1 边界定义文档

| 项目 | 内容 |
|------|------|
