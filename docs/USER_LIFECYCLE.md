# 顺时万亿产品：用户生命周期设计

> 从 18 岁到老年，让顺时陪伴用户一生

---

## 核心理念

**顺时不只是一个 App，而是一个「生活节律系统」。**

用户不需要"打开"顺时，顺时只需要在需要时出现，不需要时消失。

## 二、各年龄段用户画像

### 2.1 学生党（18-22 岁）

**场景**：大学生活、考研、实习

**核心需求**：
- 作息管理（熬夜赶论文）
- 考试压力疏解
- 饮食不规律提醒
- 简单养生习惯培养

**交互方式**：
- 轻度陪伴
- 每日一句问候
- 考前情绪支持

**产品形态**：
- 手机 App（主要）
- 微信小程序（轻量）

**收费模式**：
- 免费版足够
- 未来可选付费功能（深度学习提醒）

**示例对话**：
```
用户：最近要期末考了，睡不着
顺时：听起来压力很大。先深呼吸三次？
       考完好好睡一觉，其他的先放下。
```

## 三、跨年龄段的共同设计原则

### 3.1 退让机制（始终如一）

```
原则：顺时永远知道什么时候不出现

• 工作时间（9:00-18:00）：不主动打扰
• 深夜（23:00-7:00）：除非紧急，不发消息
• 用户忙碌时：不追问
• 用户明确拒绝时：停止
```

### 3.2 关怀分级

| 级别 | 用户状态 | 关怀频率 | 方式 |
|------|----------|----------|------|
| **基础** | 正常 | 每日 1-2 次 | 轻提醒 |
| **关注** | 偶尔情绪波动 | 每日 2-3 次 | 主动询问 |
| **深度** | 持续低落 | 每日 3-5 次 | 陪伴+建议 |
| **紧急** | 异常信号 | 立即 | 通知家人+建议求医 |

### 3.3 异常检测与响应

```
用户输入 → 风险检测 → 分级响应

风险等级：
├── 低风险（普通情绪）→ 正常陪伴
├── 中风险（持续低落）→ 关注 + 轻度跟进
├── 高风险（异常信号）→ 深度关怀 + 建议求医
└── 紧急风险（自伤倾向）→ 立即通知家人 + 紧急联系
```

## 四、用户获取与转化漏斗

```
认知 → 下载 → 激活 → 留存 → 付费 → 推荐

认知：
• App Store 推荐
• 口碑
• 健康类 KOL

下载：
• 极简下载流程
• 首日引导（1 分钟完成）

激活：
• 首日 3 条温和对话
• 了解用户基本状态

留存（关键）：
• 每周至少 1 次有价值的提醒
• 每月 1 次小结
• 永远不打扰正常工作

付费转化：
• 第 7 天：轻度引导尊享版
• 第 30 天：深度关怀提示
• 自然转化（不用促销）

推荐：
• 子女给父母开通
• 夫妻共享
```

## 七、总结：为什么能到万亿

| 维度 | 传统健康 App | 顺时 |
|------|-------------|------|
| **定位** | 工具 | 陪伴 |
| **交互** | 用户找 App | App 找用户 |
| **频率** | 需要时打开 | 时刻在线 |
| **年龄段** | 特定人群 | 全生命周期 |
| **商业模式** | 卖功能 | 卖陪伴 |
| **护城河** | 功能 | 习惯 + 情感 |

**万亿逻辑**：
- 每个人都年轻过 → 都会变老
- 健康是永恒需求
- 陪伴是永恒需求
- 顺时陪伴用户一生
